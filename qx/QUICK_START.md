## 🚀 五分钟快速入门

### 第 1 步：导入配置文件
```bash
打开 QuantumultX → 风车图标 → 配置文件 →
点击 + 按钮 → 选择 QuantumultX_Overseas.conf
```

### 第 2 步：添加节点订阅
在配置文件中找到这一行并替换为你的订阅链接：
```
[server_remote]
https://your-subscription-link.com/nodes.yaml, tag=国际节点...
```

### 第 3 步：更新规则
```
点击风车 → 更新所有 → 等待完成（约 1-2 分钟）
```

### 第 4 步：选择策略
```
点击策略 → 选择"自动选择" → 系统将自动选最快节点
```

### 第 5 步：开始使用
```
在底部点击圆形按钮启用代理 ✅ 完成！
```

## 🔧 常见自定义

### 想用美国节点代替新加坡？
编辑 `QuantumultX_Overseas.conf`，找到这一行：
```
static=虚拟币交易, 新加坡加速, 美国节点, 日本节点, direct, proxy
```
改为：
```
static=虚拟币交易, 美国节点, 新加坡加速, 日本节点, direct, proxy
```

### 想添加新的虚拟币交易所？
编辑 `CryptoFilter.list`，在末尾添加：
```
DOMAIN-SUFFIX,your-exchange.com
```

### 想禁用虚拟币加速？
在 `QuantumultX_Overseas.conf` 中找到虚拟币分流规则行：
```
https://raw.githubusercontent.com/Yeats33/net-rules/main/qx/CryptoFilter.list
```
在该行前面加 `;` 注释掉即可

### 想改变虚拟币使用的节点？
虚拟币交易策略组已配置优先级，可手动切换：
```
打开 QuantumultX → 点击"策略"标签 → 选择"虚拟币交易"
默认优先级：阿联酋 > 新加坡 > 美国 > 日本
点击切换到其他节点（重新导入配置后会恢复默认）
```

### 想调整虚拟币节点优先级？
编辑 `QuantumultX_Overseas.conf`，找到这一行：
```
static=虚拟币交易, 阿联酋节点, 新加坡加速, 美国节点, 日本节点, direct, proxy
```
根据需要调整顺序，例如优先新加坡：
```
static=虚拟币交易, 新加坡加速, 阿联酋节点, 美国节点, 日本节点, direct, proxy
```

## ⚡ 性能调优建议

### 节点测速间隔
**当前设置**：15 分钟测速一次
**如需更快**：编辑配置，改 `check-interval=900` 为更小的数字
- `check-interval=300` → 5 分钟测速
- `check-interval=600` → 10 分钟测速

### DNS 缓存
建议在 QuantumultX 设置中启用 DNS 缓存以加快解析速度

### 禁用不需要的规则
如果配置文件过大，可在 `rewrite_remote` 中注释掉不用的重写规则

## ⚠️ 注意事项

### 虚拟币交易安全提示
```
⚠️ 进行大额交易前：
  1. 确认访问的是官方网站
  2. 验证节点延迟（< 200ms 为佳）
  3. 使用"节点纯净度查询"确保节点未被篡改
  4. 了解您所在地的虚拟币政策
  5. 咨询法律和税务专业人士
```

### 虚拟币政策提醒
- 各地虚拟币政策差异大，需遵守当地法律
- 配置优先使用虚拟币政策最宽松的地区节点（阿联酋、新加坡）
- 详见 `CRYPTO_POLICY_GUIDE.md` 了解各地政策详情
- 虚拟币交易可能产生税收义务，需保留完整记录

### 性能注意
- 第一次导入配置会比较慢（正在加载数百条规则）
- 建议在 WiFi 环境下进行首次配置
- 某些规则可能与特定 APP 冲突，可按需禁用

## 🆘 故障排查

### 问题 1：导入后提示"配置无效"
**原因**：文件格式错误或路径不对
**解决**：确保使用正确的 `.conf` 文件，检查文件没有被修改

### 问题 2：虚拟币交易所仍然很慢
**原因**：节点质量差或选错了地区
**解决**：
```
1. 在策略界面手动切换节点
2. 观察不同节点的延迟
3. 选择延迟最低的节点
```

### 问题 3：某些 APP 无法访问
**原因**：被分流规则影响
**解决**：在配置中找到 `[filter_local]` 部分，检查是否有冲突规则

### 问题 4：DNS 不生效
**原因**：之前的 DNS 缓存未清除
**解决**：
```
进入系统设置 → 已连接的 VPN → 删除后重新导入配置
```
